【关键认知修正】
- 关系不是"推理"出来的, 关系本身就是属性(P3逻辑关系槽: 因果/转折/并列/递进…)。
- ABC 不是"算关系", ABC 是逻辑审查(核对对错); 数值判断与 loss 由矩阵
  (P3-L / P7 attention)算出。→ 看现有 ABC_StageA(结构决策=审查)即是此逻辑。
- 记忆/预测(E/F)= 架构师的"读写头 + 三层缓存"机制, 已实现读通路。
- 全程无外部模块(无 GNN / PINN / Mamba / 向量数据库)。

================================================================
一、第一梯队 A/B/C — token级认知 (现有升级 + 对齐loss)
================================================================
A 锚定对齐: 属性区激活识别模态(视觉[128:384]/语言[0:128]) + P3Vis空间槽对齐。
B 跨模态绑定: cross_modal_linkage(P3-L attention)绑定视觉属性↔语言属性。
C 概念组合: 抽象概念 concept(384) + ★概念对齐loss(视觉概念≈文本概念, 余弦→1)。
  → 这三层 + 对齐loss = 现在能落地能验证的"看图说词"感知通道。

================================================================
二、D — 关系审查 (不是新层, 是 ABC 审查属性关系)
================================================================
机制: 关系已在 P3 属性里(逻辑关系槽 42-52: 因果/转折/并列/条件/递进/让步…
      + 语义关系 + cross_vl 跨模态关系)。
      D = ABC(现有 StageA 结构决策的逻辑)对这些关系槽做"审查/核对",
      判断关系是否自洽(如: 视觉"树" + 文本"水果" → Is-A 自洽)。
      具体对错的数值 + loss → 由 P3-L / P7 的 attention 矩阵算。
落地: 不新增复杂 D 层; 复用 ABC 审查逻辑 + P3 关系槽 + cross_vl。
自监督(可选): Masked Concept Prediction — 遮"苹果"词, 让审查从"红+圆"属性推,
              防关系审查在小数据下空转。

================================================================
三、E — 记忆 (读写头 + 三层缓存, 已实现读通路)
================================================================
现成机制(test_context_cache.py + attr_dialogue_v2.py):
  三层缓存: GPU热层(sent_vec 50条,<1ms) → RAM温层(1万条) → Disk冷层(.json无限)
  读头: 向量cos(0.6) + P3属性Jaccard(0.4) 混合检索 top-K 历史
  写头: 当前句 → P7 sent_vec[256D] + P3属性包 → 三层FIFO存档 (无需训练)
  自适应窗口 + 跑题检测(PAOTI)
接入 V22:
  写: 每条图文样本的认知表征(sent_vec+属性)存入缓存。
  读: 处理新样本时召回相关历史 → 注入 P7 的 640D 向量(先验知识)。
  ★ 关键待接: "写通路"(检索结果 → 喂 P7 → 影响 P6 输出) —— 说明文件标注"待接入"。
无需: 外部记忆矩阵 / 向量数据库 (sent_vec 自带语义)。

================================================================
四、F — 预测决策 (读写头召回 + P6)
================================================================
机制: F 也是读写头+缓存 —— 用读头召回相关历史认知, 结合当前 A-E 的表征,
      送入现有 P6 解码(P6 本就是"预测下一个词", CE loss)。
      工业控制场景: P6 输出头也可出 Action token。
落地: 复用 E 的读写头/缓存 + 现有 P6。
无需: Mamba/RWKV 时序模块 (P6 已是预测器; 时序状态由缓存的历史序列维持)。

五、Loss
================================================================
1. 图→文 CE (主监督, 现有)。
2. ★ 概念对齐 loss @C (视觉概念≈文本概念, 让 cross_vl 不空转)。

4. E/F: 记忆召回本身不需额外 loss(读写头无需训练);
        召回的历史通过影响 P6 输出, 由主 CE loss 间接优化。
