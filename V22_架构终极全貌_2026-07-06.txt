第Ⅰ层 · 底层引擎层 — 不变的核心
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

  CharEmbed [742K]
    约3000字符 × 256D, 正交初始化。
    每字独立方向, 无余弦混淆。tied weight到P6输出。

  P3 AttributeStack [0参数, 14模块规则引擎]
    每token → 128D可解释语言属性(3层126槽位)
      Layer1(0-41): 词类(名/动/形/副/代/量/介/连/助/叹 10槽)、
                    语义(人物/地点/时间/物体/行为/状态等 14槽)、
                    句法(主/谓/宾/定/状/补/兼语/连动/同位/独立 10槽)、
                    情感(正/负/中 3槽)、时态(过去/现在/将来 3槽)、
                    人称(第一/第二/第三 2槽)
      Layer2(42-83): 逻辑关系(因果/转折/并列/条件/递进/让步/选择/承接/
                    总结/举例/目的 11槽)、修饰(形容词/方式/范围/定语/状语/
                    补语/比较 7槽)、语义格(64-75)、关联(76-83) — 部分待扩展
      Layer3(84-125): 语用/篇章/领域/风格 — 待扩展

  P7 CrossSentence [449K]
    16头 × 16D = 256总维。
    Q=A句字向量, K/V=全词表。
    RMSNorm Q/K防连乘爆炸。
    forward_batch批量处理, 层级调制(explore→meta→gate)。
    输出: word_out[nA,256] + sent_vec[256]。

  P6 SentToWords CE [8.6M, tied weights]
    encoder(256→256→256, Gate调制) → 128头 × Linear(256,256)
    → word_vec × char_embed^T → vocab logits → CE直出字。
    tied weights: 输入embedding = 输出embedding, 参数从~40M降到8.6M。

第Ⅱ层 · 认知栈 — ABCDEF 六层逻辑(核心)
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

  [数据流]
    char_emb(256) + attr(128) = 384D → p7_input_proj (V22: 640→256)
    → P7 → word_out(256) + sent_vec(256)
    → P3-L(属性关联) → ABC'(内容加压) → ABCDEF → Gate → P6

  [维度总纲 — V22兼容扩展]
    语言属性层: 128D (P3)
    视觉属性层: 256D (P3Vis, 槽位128-383)
    全属性向量: 384D (语言+视觉)
    Token输入: char_emb(256)+全属性(384)=640D → p7_input_proj(640→256)
    StageA输入: 640D → 256隐层 → 20类结构
    StageB输入: 128(ah) + 384(attr_sum) + 128(p3lf) = 640
    StageC输入: 384(content) + 5(情感)
    StageD输入: 384(content) → 11类关系
    StageE输入: 384(rd) → 384(me)

  Stage A — 结构决策 [77K, in_dim=640, n_classes=20]
    输入: attr(384)∥word_out(256)=640D
    结构: encoder(640→256→128) + pool(128→128) + classifier(128→20)
    输出: a_logits[20] + ah[128]
    20类 = 14语言P3子模块 + 5视觉大类(色彩/形状/纹理/几何/空间) + 1通用(情感基)

  Stage B — 内容填充 [36K, content_dim=384]
    输入: ah(128) + attr_sum(384) + p3lf(128) = 640D
    结构: fuse(640→256→128→384)
    输出: ct[384] — 跨模态概念内容

  Stage C — 语气精炼 [23K, content_dim=384]
    输入: ct(384) + 情感5D
    结构: refine(389→96→→48)
    输出: ac[48] — 语气控制信号, 软归一化(ac/norm(ac)+1.0)

  Stage D — 关系审查 [153K]
    输入: ct(384)
    结构: encoder(384→384→384) → rel_head(384→11) + rd(384, 球面归一化)
    输出: rel_logits[11] + rd[384]
    11类关系: 因果/转折/并列/条件/递进/让步/选择/承接/总结/举例/目的
    关系=属性(P3逻辑关系槽42-52), ABC审查其自洽性, 不引入GNN。
    rd球面归一化前向受控 + 反向不冲击, 可全量注入gate_input(384→512, 待gate分组重构)

  Stage E — 记忆读写头 [178K]
    输入: rd(384) + [历史记忆中检索的me_recall]
    结构: net(384→384→384) → 球面归一化 → me(384)
    融合: me = F.normalize(net(rd) + recall × 0.1, dim=-1)
    输出: me(384) — 加法注入P6解码(sv_rich = sv + abc + me×0.1)
    读写机制:
      WRITE — sent_vec(256) + me(384) → FIFO记忆库(上限2000条, 落文件V22_mem.pt)
      READ  — query_sent_vec余弦检索top-1 → P3属性Jaccard过滤(权重0.4)
              → 召回历史me为detach先验, 不回传梯度 → 球面融合注入
    三层缓存: GPU热层(50条, <1ms, 余弦检索)
              RAM温层(10000条, ~0.1ms, 属性Jaccard过滤)
              Disk冷层(无限, JSON文件, 按需加载, sent_vec自带语义无需外部embedding)
    关键设计:
      - 写头无需训练(P7+P3产出sent_vec+属性包, 不依赖外部模型)
      - 读到注入全程detach, 记忆是锚点不参与反向传播
      - 同一个me向量(fused)既注入P6增强解码, 又作为下一个token的上下文写入记忆库
      - 球面归一化防数值冲击: F.normalize(me) + F.normalize(recall_me), 全量融合不崩

  Stage F — 预测解码
    整合A-E五层认知表征 → P6 128头解码器 → CE损失直出字
    不引入新的自回归模块, 直接复用P6的decoder架构

  cross_vl 跨模态关联 (P3-L中)
    视觉离散值(vis_hue/vis_shape/vis_texture) → register_from_vision → 视觉属性ID
    → cross_modal_linkage(语言ID↔视觉ID, 双向注意力) → 关联分数 → 序列场景中建立"文字↔图像"硬匹配

第Ⅲ层 · 控制与训练面板
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

  ABC' 内容加压 [86K]
    按P3-L关联分布(p3lf, 128D) → 128→192→128小网络 →
    abc_prime(128)
    冷启动: 模块gain=0.01, bias=0, 不破坏训练
    双路径: ①注入ABC输入端(p3lf+abc_prime×0.1) 提升控制质量
             ②加法注入P6输入端(sv + abc_to_sent(abc_prime)×0.1) 增强解码

  Gate三路独立门控 [939K]
    三路信号(各自独立通道, 加性融合):
      base: gate_base_proj(abc_conf, 12→128) — ABC置信度统计
      abc:  abc_to_gate(ac_batch, 48→128)   — ABC语气控制
      sent: sent_to_gate(sv_batch, 256→128)  — P7语义信号
    拼接: [base 128 ∥ abc 128 ∥ sent 128] ∥ rd_to_gate(rd 384→128, 预留)
    → TurboExplore(384→512→512→256) → TurboMeta_vMF(dim=256, μ+κ可学习, 球面偏好方向)
    → vMF球面门控: gate_i = sigmoid(κ_i · cos_sim(x_norm, μ_i) + bias_i)
	    → gates[256] → 调制P6 encoder (κ可学习, 球面拓扑内化)

  Master Gate [0.4K]
    5-6子损失聚合 → 动态温控(0-1) → 各损失自动平衡
    V22: 因loss_D冷启动跳过(gate崩溃导致尖峰→CE=5.0→全网梯度死亡)

  7独立Adam优化器 + 分层学习率
    opt_gate:    ×2.0(base=0.003)  — 控制线需极快响应
    opt_p7:      ×0.05             — 严格压制(爆炸源头)
    opt_abc:     ×1.2              — 参数少, 需敏捷
    opt_abc_d/e: 位于opt_abc组     — D/E用独立球面归一化防梯度冲突
    opt_p6:      ×1.0              — 标准
    opt_p3l:     ×0.8
    opt_master:  ×0.5

  5层智能防护系统
    ① ac_batch实时检测 → 超ac_ema×3触发冷却(早于CE检测)
    ② CE单次尖峰 → 超EMA×3或绝对值>5触发clamp+cool_down=50
    ③ aux_scale渐进自愈 → 跌:0.95/步(快速响应), 涨:1.01/步(~70步全恢复)
    ④ NaN自动回滚 → 逐模块参数NaN检测 → 从best.pt恢复 → 跳过本批step
    ⑤ loss<0.0001 → 自动停车

  ★ 球面归一化(关键修复)
    问题: D/E层注入后训练20-160不定崩溃 — CE→5.0, explorer/meta/gate全梯度归零,
          abcB的grad/param比爆至5.87(正常<0.05)。
    崩溃链: D/E输出数值范围过大 → ac_batch/gate_input膨胀
            → meta sigmoid饱和 → gate梯度死 → 连锁全网崩
    误判: ①冷启动gain(0.1→0.01)只推后不治本(权重训大照样冲)
           ②回退gate(512→384)仍崩 — 根因不在通道宽度, 在数值范围
    修复: rd/me/像素/FFT全部F.normalize(dim=-1) → 球面单位向量
          前向数值受控 + 反向梯度不冲击 → 全量注入(×0.1)不崩
    教训: "数值范围太大梯度承受不住"是根因; 球面归一化控数值(同ac_batch
          软归一化、P3-L F.normalize一个哲学)

第Ⅳ层 · 多模态扩展(统一token = Embed球面256 + Attr384)
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

  原则: P7/ABCDEF/P6不分token来源, 拒绝ViT/Whisper式的每模态独立模型。

  ┌──────────┬─────────────────────┬───────────────────────────┐
  │ 模态     │ Embed 256D(球面)    │ Attribute                  │
  ├──────────┼─────────────────────┼───────────────────────────┤
  │ 文本     │ char_embed(字)      │ P3 语言属性(128D)          │
  │ 视觉     │ VisualEmbed(像素)   │ P3Vis(256D, 128-383槽位)  │
  │ 声学     │ AudioEmbed(FFT谱)   │ P3Aud(预留, 384-639槽位)  │
  └──────────┴─────────────────────┴───────────────────────────┘

  VisualEmbed [197K]
    16×16原始RGB像素 = 768数字(÷255归一化) → Linear(768,256)
    → 正交初始化 → F.normalize球面(单位向量)
    无CNN/ViT/预训练黑盒, 与char_embed同哲学。

  AudioEmbed [131K]
    原始FFT幅度谱(512频点, 最底层无梅尔滤波) → Linear(512,256)
    → 正交初始化 → F.normalize球面(单位向量)
    拒绝梅尔/MFCC(引入人为听觉先验), 保持数据物理完整性。

  P3Vis 工业级视觉属性 [0参数, 5层规则引擎]
    256D白盒可解释视觉特征:
      V1 色彩物理(48槽): HSV联合直方图(16×8), 曝光(高光/暗部/动态范围),
          色温白平衡, K-means Top-4主色块(4×4), 色彩心理(冷暖占比)
      V2 几何形态(64槽): 前景/bbox, Canny边缘密度, HOG 8方向,
          Hu 7不变矩(对数标度), 连通域数/孔洞/欧拉数(拓扑)
      V3 纹理材质(48槽): GLCM(对比度/能量/同质性/相关性×4方向),
          LBP均匀直方图, 材质反光/粗糙度代理, 频域周期性(FFT)
      V4 空间深度(48槽): 九宫格方位one-hot, 质心坐标,
          显著性(谱残差法, 需opencv-contrib; 无则留空)
      V5 时序语义(48槽): 场景复杂度(边缘密度), 运动/光流(预留多帧),
          跨模态锚点(预留OCR/标志)
    全部为传统CV规则(非深度学习), 连续特征不归一化(由控制层自适应)。

  P3Aud 声学属性 [预留, 当前模拟验证用零值]
    基频/共振峰/响度/过零率/频谱质心/节奏 — 对应P3Vis的同构属性提取
