═══════════════════════════════════════════════════════════════
一、大一统哲学: 万物皆 Token, 信号层统一
═══════════════════════════════════════════════════════════════

  拒绝"每模态一套特化模型"(Whisper/ViT/Wave2Vec = 特化烟囱),
  三模态统一成: 底层信号 → Embed(256D 球面) + 属性(384D) → 640D 同构 token
              → 一套 P7 → ABCDEF → P6

  ┌────────┬──────────────────────────┬────────────────────┐
  │ 模态   │ Embedding(256D 球面)     │ 属性区              │
  ├────────┼──────────────────────────┼────────────────────┤
  │ 文本   │ char_embed(字符整数)     │ attr[0:128] 语言    │
  │ 视觉   │ VisualEmbed(16×16像素768)│ attr[128:384] P3Vis │
  │ 声学   │ AudioEmbed(FFT幅度谱)    │ attr[384:640] P3Aud*│
  └────────┴──────────────────────────┴────────────────────┘
  (* 声学独立属性区 attr[384:640] 为完全同构版, 属工程落地待做;
     当前声学先验证信号流通, attr 暂 0)

  关键: P7/ABCDEF/P6 分不出 token 是字/像素/频率 —— 这才是通用引擎。
        未来触觉/嗅觉/脑电波, 同样"信号→球面Embed+属性", 一行架构不改。

═══════════════════════════════════════════════════════════════
二、三模态底层信号 (坚持最底层, 拒绝人为先验)
═══════════════════════════════════════════════════════════════

  文本: 字符整数 → char_embed 正交初始化 (V21 原样)

  视觉: 16×16 原始 RGB 像素 = 768 数字 → Linear(768,256) → F.normalize 球面
        — 用原始像素, 不用 CNN/ViT 特征图 (那是黑盒预训练污染)

  声学: 原始 FFT 幅度谱(512 频点) → Linear(512,256) → F.normalize 球面
        — 用最底层 FFT, 不用梅尔频谱/MFCC
        理由: 梅尔 = 预设"人耳听觉模型"(等响曲线) = 人为先验污染;
             FFT = 纯物理频率分解, 无任何假设, 和视觉原始像素完全对称。
        让 P3Aud 自己学"什么频率重要", 不用 mel 滤波器替模型做决定。

═══════════════════════════════════════════════════════════════
三、ABCDEF 六层认知堆栈 (在 V21 的 ABC 三阶上扩)
═══════════════════════════════════════════════════════════════

  A 结构决策 (V21): attr+word_out=640 → 20类结构 → ah[128]
  B 内容填充 (V21): ah+attr_sum+p3lf → ct[384]  (V22: content 128→384)
  C 语气精炼 (V21): ct+情感 → ac[48]
  ─────────────── V22 新增 D/E ───────────────
  D 关系审查 (V22): 关系即属性(P3逻辑关系槽42-52), ABC审查其自洽性,
                    不引入 GNN。ct(384) → 关系表征 rd(384, 球面) + 关系类logits(11)
  E 记忆读写头(V22): sent_vec 余弦检索历史 me → 球面融合;
                    写头存(sent_vec, me)入记忆库(FIFO 2000, 落文件 V22_mem.pt);
                    rd(384) → me(384, 球面), 记忆是 detach 先验不回传
  F 预测解码: 复用现有 P6 (P6 本就是"预测下一个词", 无需 Mamba 时序模块)

  维度联动(V22 扩): 属性 128→384, p7_input_proj 384→640,
                    StageA in_dim 640, StageB fuse/content 384, p3l attr_dim 384

═══════════════════════════════════════════════════════════════
四、★ 球面归一化: 数值范围控制 (V22 最关键突破)
═══════════════════════════════════════════════════════════════

  问题: D/E 层加入后, 训练 E20-160 不定崩溃 ——
        CE 突回 5.0, explore/meta/gate 全 grad=0, abcB ratio 爆到 5.87(正常<0.05)

  崩溃链(按 bug 记录追出):
    D/E 输出数值范围过大 → 注入撑大 ac_batch/gate_input
    → meta 的 sigmoid 饱和 → gate 梯度死 → 连锁全网崩

  误判(两次):
    ① 以为注入系数大 → 冷启动 gain 0.1→0.01 → 无效(只推后崩溃)
    ② 以为 gate 窄桥(explore 384→512) → 回退 → 仍崩

  根治: rd/me/像素/FFT 全部 F.normalize(dim=-1) → 单位球面向量
        前向数值受控 + 反向梯度不冲击 → 可全量注入(0.1)不崩
    教训: 新模块注入不是缩系数(冷启动只推后, 权重训大了照样冲),
          而是球面归一化控数值范围 —— 同 V21 的 ac_batch 软归一化、
          P3-L 的 F.normalize 一个哲学。"数值范围太大梯度承受不住"是根因。

═══════════════════════════════════════════════════════════════
五、跨模态对比 loss 的方向性 (架构师第一性原理)
═══════════════════════════════════════════════════════════════

  图→文 / 声→文: 余弦相似度 cos(信号权重, 文权重)
    — 只需"方向对得上"(找语义匹配的字)

  文→图 / 文→声: 皮尔逊相关系数 Pearson(文权重, 信号)
    — 生成信号要"数值线性对应", 走特殊通道(跟属性逻辑处理做一个通道)

  当前: 图→文/声→文的余弦已跑通(硬编码); 文→信号的皮尔逊通道待落地。

六、三模态验证结果 (--sample 10 文本 + 视觉28对 + 声学模拟)
═══════════════════════════════════════════════════════════════

  文本: loss 6.98 → 0.0001 自动停训, 不崩 (球面归一化 + ABCDE 稳定)
  视觉: 图→文 loss 3.00→1.01, "图(苹果) → 预测: 苹果" 完全正确 ✅
  声学: 声→文 loss 1.83→1.24, "声(FFT) → 预测: 高音" 出字(数据流通) ✅
        (未区分低/中/高 = 训练不足30轮+模拟数据简单+逻辑未训, 非架构问题)

  结论: 一套 ABCDEF 引擎, 文本/视觉/声学三模态数据全流通。

七、V21 → V22 演进 (扩展, 不是重构)
═══════════════════════════════════════════════════════════════

  V21 (6/27): 单模态文本, 12模块 9.9M, ABC三阶, 128D属性
  V22 (7/05): 三模态统一, ABCDEF六层, 384D属性(语言+视觉),
              +VisualEmbed/AudioEmbed球面, +球面归一化数值控制,
              +E记忆读写头文件缓存, +跨模态余弦/皮尔逊对比

  核心教训(反复强调): V21 是通用 token 引擎, 扩多模态只是
    P3/ABC/维度/loss 的"扩展", 不是重构; 别把扩展做成任务特化的
    mini 验证(接不进正式架构 = 白做 + 烧上下文注意力)。
